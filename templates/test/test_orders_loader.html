Test du Loader - Section Commandes

🐛 Problème Identifié

Le logo de chargement dans la section "Commandes" ne s'affichait pas correctement.

  • ❌ Le loader row n'était pas visible (display: none)
  • ❌ Mauvais type d'affichage pour les table rows
  • ❌ Animation ne démarrait pas correctement

✅ Solution Appliquée

Correction complète du système de loader pour la section Commandes:

  • ✅ CSS: Ajout de classes .show/.hide pour les table rows
  • ✅ JavaScript: Détection du type d'élément (TR vs autres)
  • ✅ Animation: Reset et démarrage correct de l'animation
  • ✅ Consistence: Même comportement que les autres sections

🔧 Modifications Techniques

Fichiers modifiés:

  • static/css/admin.css - Classes .show/.hide pour les loader rows
  • static/js/admin.js - Fonctions show/hideSectionDataLoader améliorées

Changements clés:

  1. Détection du type d'élément (TR vs autres conteneurs)
  2. Utilisation de classes CSS au lieu de styles inline
  3. Reset d'animation avant démarrage
  4. Délai de 100ms pour démarrage fluide

🚀 Vérification en Production

Pour vérifier que la correction fonctionne:

  1. Ouvrir l'interface admin
  2. Naviguer vers "Commandes"
  3. Le loader animé doit apparaître immédiatement
  4. Après chargement, les données doivent s'afficher
  5. Le loader doit disparaître proprement
